Querying Wolfram|Alpha

The =wolf query command is used to query Wolfram|Alpha. query can be anything that you'd be able to put into Wolfram|Alpha normally. Due to both the speed of W|A and the amount of data that gets send to Discord, this command can be slow, so please be patient.

Examples:

  • =wolf intersection of sin(x) and cos(x)
  • =wolf plot x^3 - x^2 + x - 1
  • =wolf dark blue

Formatting equations with LaTeX

The =tex equation command is used render LaTeX equations. equation can be any valid LaTeX equation.

Examples:

  • =tex x = 7
  • =tex \sqrt{a^2 + b^2} = c
  • =tex \int_0^{2\pi} \sin{(4\theta)} \mathrm{d}\theta

Notes

The there are a few non-standard shortcuts to make your life easier. Parenthesis ( and ) get \left and \right added to them automatically. In a few cases, the bot will recognize malformed LaTeX, and will tell you how to correct it. These features can be disabled by using the the =rtex command instead.

Rendering is done by QuickLatex.com.

Limitations

The bot currently uses an external rendering service. As such, the following quirks should be acknowledged.

  • $ to start and end equations are not required, and may do strange things. Use \text{words} to place text.
  • \usepackage, will not work.
  • Macros such as \@ifpackageloaded will probably fail.
  • Loading images is not officially supported. It might work but probably won't.

Using the Calculator

The calculator can be invoked with either =calc equation or ==equation.

Examples:

  • =calc 2 ^ (1 + 3) * 5
  • =calc round(10 / 3)
  • =sort 8d6 : 10

Operations

  • + : addition
  • - : subtraction
  • * : multiplication
  • / : division
  • ^ : exponentiation (has some limitations to prevent exploits)

Constants

  • pi : 3.141592...
  • e : 2.178281...

Functions

  • round : rounds a number to the nearest integer
  • int : rounds *down* to the nearest integer
  • sin : sine of an angle (in radians)
  • cos : cosine of an angle (in radians)
  • tan : tan of an angle (in radians)
  • deg : converts radians to degrees
  • rad : converts degrees to radians

Dice Rolling

  • dX : rolls an X-sided die. (e.g. d20)
  • YdX : rolls Y X-sided dice and gives the total. (e.g. 2d6)

You cannot roll more than 1000 dice at once.

Repeating Equations

Appending the equation with : R, where R is an integer will repeat the calculation that many times. This is only useful only if you want to roll many dice at once. Using the =csort or =sort command will sort the numbers in increasing order. R may not be greater than 50.

Settings and Management

MathBot allows you to assign permissions on who may use various commands and where they may be used. Settings are currently considered experimental. Any bugs or issues should be reported on the official server.

Note: role-based permissions are still in development.

Command Syntax

The general structure of the settings command is:

=set context setting value

  • context should be either server to apply the setting to the whole server, or channel, to apply the setting to the current channel. These can be shortened to s and c respectively.
  • setting should be the name of once of the settings, listed below.
  • value should be either enable, disable, or original. enable will allow people to use the command in the given context and disable will stop people from using it. original will revert the setting to its default value.

List of Settings

  • c-tex : Enable or disable the =tex command. Default: Enabled
  • c-calc : Enable or disable the =calc command. Default: Enabled
  • c-wolf : Enable or disable the =wolf command. Default: Enabled

Setting Priorities

Channel-specific settings always override the server setting. If neither is specified, the default value is used.

Examples

Disable the =wolf command on a server:

=set server c-wolf disable

Enable the =calc command on a single channel:

=set channel c-calc enable
